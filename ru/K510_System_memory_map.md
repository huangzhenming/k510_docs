# 1 Планирование системной памяти K510

План памяти K510 показан на следующем рисунке:

![](../zh/images/system_memory_map/k510-system-memory-map.png)

На справочной плате K510 crb имеется 512 МБ DDR, в общей сложности запланировано четыре области:

- Планирование 0 ~ 240M для ядра Linux
- 240M ~ 496MB планируется для общей памяти, используя метод резервирования пула памяти CMA, так что подсистема управления памятью ядра Linux также может выделять память из пула CMA при отсутствии использования Share memory
- 496M ~ 510M планируется для использования DSP
- 510M ~ 512M планируется для использования логотипа

# 2 Описание дерева устройств

Планирование памяти K510 описывается в зарезервированной манере, через узлы зарезервированной памяти дерева устройств. Соответствующая информация об узле дерева устройств выглядит следующим образом:

```text
ddr_memory: memory@0 {
    status              = "okay";
    device_type         = "memory";
    reg                 = <0x0 0x00000000 0x0 0x20000000>;
};

sharem_cma:sharem_cma@8000000 {
    compatible          = "k510-share-memory-cma";
    reg                 = <0x0 0xf000000 0x0 0x10000000>;  /*240M~496M*/
};

reserved-memory {
    #address-cells = <2>;
    #size-cells = <2>;
    ranges;

    cma_buffer: buffer@f000000 {
        compatible = "shared-dma-pool";
        reusable;
        linux,cma-default;
        reg = <0x0 0xf000000 0x0 0x10000000>;
    };

    dsp_buffer: buffer@1f000000 {
        no-map;
        reg = <0x0 0x1f000000 0x0 0xe00000>;
    };

    logo_buffer: buffer@1fe00000 {
        no-map;
        reg = <0x0 0x1fe00000 0x0 0x200000>;
    };
};
```

# 3 Конфигурация, связанная с Buildroot

План системной памяти описан в dts ядра Linux, но адрес загрузки встроенного ПО dsp должен быть настроен в buildroot:

конфигурации/k510_crb_lp3_v0_1_defconfig:

конфигурации/k510_crb_lp3_v1_2_defconfig:

BR2_TARGET_EVB_FIRMWARE_LOAD_ADD=0x1f000000
